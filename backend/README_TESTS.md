## Структура тестов

### Файлы тестов

- `tests/controllers.test.js` - юнит-тесты для контроллеров (Auth, Orders, Menu, CoffeeShops)
- `tests/services.test.js` - тесты для сервиса Telegram (парсинг initData, верификация хэша)
- `tests/middleware.test.js` - тесты middleware аутентификации JWT

### Типы тестов

**Контроллеры** - тестируют обработку запросов с моковыми данными базы данных:
- Проверка валидации входных данных
- Проверка ответов при различных сценариях (успех, ошибки 400, 401, 404)
- Проверка вызовов базы данных с правильными параметрами

**Сервисы** - тестируют бизнес-логику:
- `parseInitData` - парсинг строки initData из Telegram WebApp
- `verifyTelegramHash` - криптографическая верификация целостности данных

**Middleware** - тестируют защиту маршрутов:
- Проверка наличия и формата Authorization header
- Верификация JWT токена
- Установка user в запрос при успешной аутентификации

## Используемые инструменты

- **Jest** - фреймворк для тестирования
- **Supertest** - для функционального тестирования HTTP (подготовлен)
- **Babel** - поддержка ES modules в Jest

## Моки

Тесты используют моки для внешних зависимостей:
- `pool.query` - моковая база данных PostgreSQL
- `bcryptjs.compare` - моковый bcrypt (для тестов auth)
- `jsonwebtoken` - моковый JWT (для тестов middleware)

## Команды

```bash
# Запуск всех тестов

# Запуск с покрытием
npm test -- --coverage

# Запуск конкретного файла тестов
npm test -- tests/controllers.test.js

# Запуск в watch режиме
```
