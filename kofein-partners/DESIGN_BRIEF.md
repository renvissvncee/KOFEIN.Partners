---

## 📋 О проекте

**KOFEIN.PARTNERS** — веб-приложение для управления кофейней. Предназначено для двух ролей:
- **Сотрудник** — просмотр и обработка заказов
- **Владелец** — управление меню, заказами и настройками кофейни

**Стек технологий:**
- Frontend: React 19 + Vite + TailwindCSS v4
- Backend: Node.js + Express + PostgreSQL
- Иконки: lucide-react

## 📱 Структура страниц

### 1. App.jsx (главный экран)
- **Хедер:** Логотип (буква K в квадрате), переключатель темы, кнопки ролей
- **Основной контент:** EmployeeOrders или OwnerPanel
- **Футер:** Копирайт с версией

### 2. EmployeeOrders.jsx (экран сотрудника)
**Функционал:**
- Просмотр списка заказов
- Фильтрация по статусу (Все / Новые / Готовятся / Готовые / Закрытые)
- Просмотр деталей заказа
- Изменение статуса заказа (New → Preparing → Ready → Closed)

**Компоненты:**
- Список карточек заказов (3 колонки на десктопе)
- Фильтры-кнопки с счётчиками
- Детальный вид заказа при клике
- Кнопки действий в зависимости от статуса

### 3. OwnerPanel.jsx (экран владельца)
**Функционал:**
- Управление меню (CRUD товаров)
- Просмотр всех заказов
- Настройки кофейни (название, адрес)

**Вкладки:**
1. **Меню** — список товаров, добавление, редактирование, удаление, доступность
2. **Заказы** — история заказов с фильтрацией
3. **Настройки** — форма редактирования кофейни

## 🌐 API Endpoints

### Orders
- `GET /orders?coffee_shop_id={id}&status={status}` — список заказов
- `PUT /orders/:id/status` — обновление статуса

### Menu
- `GET /menu?coffee_shop_id={id}` — список товаров
- `POST /menu` — добавить товар
- `PUT /menu/:id` — обновить товар
- `DELETE /menu/:id` — удалить товар

### Coffee Shops
- `GET /coffee-shops/:id` — данные кофейни
- `PUT /coffee-shops/:id` — обновить кофейню

## 🎨 Что нужно сохранить

### Обязательно
1. **Функциональность** — не менять логику работы
2. **Цветовые акценты** — cyan (#22d3ee) и yellow (#fbbf24) остаются ключевыми
3. **Адаптивность** — должен работать на мобильных
4. **Доступность** — контрастность текста, читаемость

### Можно улучшить
1. **Типографика** — шрифты, размеры, межстрочные интервалы
2. **Пространство** — отступы, padding, margin
3. **Анимации** — плавность, переходы, микро-взаимодействия
4. **Иконки** — замена на другие стили
5. **Формы компонентов** — скругления, тени, границы
6. **Градиенты** — могут быть другими
7. **Тёмная/светлая тема** — могут быть разными цветами

---

## 🚫 Чего НЕ делать

- Не менять названия кнопок и тексты интерфейса
- Не удалять функционал (фильтры, вкладки, формы)
- Не менять структуру данных (заказы, товары)
- Не убирать адаптивность
- Не делать дизайн слишком минималистичным (нужна информативность)

## 🎯 Критерии успеха

Дизайн считается хорошим, если:
1. ✅ Все функции работают
2. ✅ Текст читаем (контраст ≥ 4.5:1)
3. ✅ Мобильная версия удобна
4. ✅ Нет пустых мест (заполнен экран)
5. ✅ Визуальная иерархия понятна
6. ✅ Цвета гармоничны
7. ✅ Анимации не мешают
8. ✅ Выглядит современно

## 📝 Примечания для дизайнера

> "Мне нужен **красивый, современный дизайн** с тёмными тонами.  
> Сохрани всю логику работы, но сделай визуально привлекательно.  
> Не оставляй пустых мест — заполни экран контентом или декоративными элементами.  
> Адаптивность обязательна (mobile-first).  
> Используй градиенты, тени, анимации для глубины."
